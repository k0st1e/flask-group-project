## Introduction. Read this if it is your first time.

An issue ca by anything. From a bug report or something that you thought of and want to request a feature that does
not exist yet. The first step to participate in this community would be to contrubute by issuing a bug report or a new feature
that you would like to see implemented. Bug reports are especially helpful to the people developing the project as they can
quickly fix the problem that is present in the program. Other than bug reports, feature requests are also vital. New ideas can move the product forward. 

## Note before submitting an issue!

As every personal computer is different please remember that we, the developer do not have direct access to your 
technology and as such it is of high importance when submitting an issue to include all the details including your 
personal setup that you are using to access this software.

## Issues Policy

If the issue is vague and not clear it will unfortunatel be closed! If it ends up being close don't hesitate and please
re-open it with additional information provided.

## Before opening Issues

Kind reminder to search before opening a new issue. Search existing issues in the project before filing a new one as
it might have already been raised by another user. When opening issues please be brief and to the point with details
that are useful to other people in order for the issue to be solved as quickly as possible.
